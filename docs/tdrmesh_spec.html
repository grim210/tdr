TDRMesh Specification

Root Object

Every TDRMesh JSON file is an object. When you open the file, the first thing you should see is an opening curly brace. Everything contained within those two braces are part of the same TDRMesh object.

Sub-Objects

From the root object, you will have other objects identifed by strings. As an example, you may see: 

{
    "example_object": {
        ...
    },
    ...,
    ...,
}
which would identify that sub-object by the name "example_object". That mesh object can contain whatever data is relevant to that structure. However, all data must be provided with a valid identifer.

  • Textures: Paths of where to find textures, and what type of format that the texture is stored as.
  • Shaders: Paths that point to where to find various shaders used for rendering the object.
  • Primitives: Typically raw float data associated with vertex or UV data, as an example.

Textures

Not a texture, but a texture path where the engine knows to look for the texture. Each texture is an array that tells you the type of texture and the path to the texture:

["DDS", "texture.dds"]

Tells you that the path is "texture.dds", and the format is "DDS", which is a DirectDrawSurface. At this point, the supported texture formats are:

  • DirectDrawSurface (DDS)

Shaders

Shaders, much like textures, aren't actually contained withint the TDRMesh json file. It's simply a path. You specify the type and the path and the TDRMesh loader will know what to do.

  • Fragment Shaders: fshader
  • Geometry Shaders: gshader
  • Vertex Shaders: vshader

As an example, this tells you that the shader you want to load is a Vertex shader, and is called "some_shader.vs":

    vshader = 'some_shader.vs'
